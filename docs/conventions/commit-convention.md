# 커밋 컨벤션 (Commit Convention)

커밋 메시지는 **태그: 제목**, **본문**, **꼬릿말**의 형태로 구성합니다.

```
태그: 제목

본문

(선택적) 꼬릿말
```

## 1. 태그 (Tag)

| 태그 | 설명 |
| :--- | :--- |
| `Feat` | 새로운 기능 추가 |
| `Fix` | 버그 수정 |
| `Refactor` | 리팩토링 (기능 변경 없음) |
| `Design` | CSS 등 사용자 UI 디자인 변경 |
| `Comment` | 필요한 주석 추가 및 변경 |
| `Style` | 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우 |
| `Test` | 테스트(테스트 코드 추가, 수정, 삭제, 비즈니스 로직에 변경이 없는 경우) |
| `Chore` | 위에 걸리지 않는 기타 변경사항 (빌드 스크립트 수정, assets, 패키지 매니저 등) |
| `Init` | 프로젝트 초기 생성 |
| `Rename` | 파일 혹은 폴더명 수정하거나 옮기는 경우 |
| `Remove` | 파일을 삭제하는 작업만 수행하는 경우 |
| `Docs` | 문서 작업 |

## 2. 제목 (Subject)

1.  제목은 최대 **50글자**를 넘지 않아야 합니다.
2.  마침표 및 특수기호를 사용하지 않습니다.
3.  영문 작성 시 첫 글자는 **대문자**를 사용하고, **명령문**으로 작성합니다. (예: `Add login feature` O, `Added` X)
4.  **개조식 구문**으로 작성하여 간결하고 요점적으로 서술합니다.

## 3. 본문 (Body)

1.  한 줄당 **72자** 내로 작성합니다.
2.  최대한 상세히 작성합니다.
3.  **어떻게** 보다는 **'무엇을', '왜'** 변경했는지에 초점을 맞춥니다.

## 4. 꼬릿말 (Footer)

꼬릿말은 선택사항이며, 이슈 트래커 ID를 참조할 때 사용합니다.

1.  형식: `유형: #이슈번호`
2.  여러 개의 이슈 번호는 쉼표(`,`)로 구분합니다.

| 유형 | 설명 |
| :--- | :--- |
| `Fixes` | 이슈 수정 중 (아직 해결되지 않은 경우) |
| `Resolves` | 이슈를 해결한 경우 |
| `Ref` | 참조할 이슈가 있을 때 사용 |
| `Related to` | 해당 커밋에 관련된 이슈 번호 (아직 해결되지 않은 경우) |
