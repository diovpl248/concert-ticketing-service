# E2E 테스트 실행 가이드 (Playwright)

이 폴더(`frontend/e2e`)에는 통합 예약/승인 시나리오에 대한 E2E (End-to-End) 테스트가 포함되어 있습니다.

## 📌 테스트 실행 조건
1. **서버 구동 상태**: 
   - 프론트엔드(`localhost:3000`) 서버가 켜져 있어야 합니다. (`npm run dev`)
   - 백엔드(Spring Boot) API 서버와 로컬 DB/Redis가 구동되어 있어야 합니다.
2. **사전 데이터 존재 여부**: 
   - 사용자가 로그인되어 있거나 자동으로 토큰이 지급되는 환경이어야 합니다.
   - DB상에 예약 가능한 **콘서트 일정 및 좌석** 데이터가 사전에 생성되어 있어야 합니다. 화면에서 클릭할 요소가 렌더링되지 않으면 에러가 발생합니다.

---

## 🚀 테스트 실행 명령어

프론트엔드 홈 폴더(`frontend`) 혹은 상위 터미널에서 다음 명령어를 실행합니다.

### 1. 백그라운드 구동 (기본 테스트)
```bash
npm run test:e2e
# 또는
npx playwright test
```
- 결과를 터미널 상에 보여주고, 문제가 있으면 에러 로그를 출력합니다.
- 완료 후 자동으로 HTML 리포트를 생성합니다.

### 2. 브라우저 확인(UI 모드)하며 디버깅하기 (추천 ⭐️)
```bash
npx playwright test --ui
```
- 전용 Playwright UI 프로그램이 실행됩니다. 재생 버튼(▶️)을 누르면 실제 브라우저가 화면상에 떠서 봇이 클릭하는 과정을 실시간으로 보실 수 있습니다.

### 3. 헤드풀(브라우저를 직접 띄워서) 로 단일 실행하기
```bash
npx playwright test --headed
```
